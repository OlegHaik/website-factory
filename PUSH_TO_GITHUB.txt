STEP-BY-STEP: Push to GitHub and Deploy to Vercel
===============================================

STEP 1: Create GitHub Repository
---------------------------------
1. Go to: https://github.com/new
2. Repository name: WATER_DAMAGE_LOCATION_WEBSITE_FACTORY
3. Make it PRIVATE (important!)
4. Don't initialize with README
5. Click "Create repository"

STEP 2: Push Your Code
-----------------------
After creating the repo, run these commands in your terminal:

cd "d:\roofing-multi-location-website-main"

git remote add origin https://github.com/HSI-Pro/WATER_DAMAGE_LOCATION_WEBSITE_FACTORY.git
git branch -M main
git push -u origin main

(Replace YOUR-USERNAME with your actual GitHub username)

STEP 3: Deploy to Vercel
-------------------------
1. Go to: https://vercel.com
2. Click "Add New Project"
3. Import your GitHub repository: WATER_DAMAGE_LOCATION_WEBSITE_FACTORY
4. Before clicking Deploy, add these Environment Variables:

   NEXT_PUBLIC_SUPABASE_URL
   (Project Settings â†’ API â†’ Project URL)

   NEXT_PUBLIC_SUPABASE_ANON_KEY
   (Project Settings â†’ API / API Keys â†’ Publishable / anon key)

5. Click "Deploy"
6. Wait 2-3 minutes
7. Your site will be live at: https://your-project.vercel.app

STEP 4: Update Supabase Domain
-------------------------------
After deployment, ensure your `sites` table contains the correct domain in `domain_url`
for the site you want to serve.
